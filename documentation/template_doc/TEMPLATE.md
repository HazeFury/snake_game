# 42 Python Project Template

## 📖 Presentation
This repository serves as a robust, modern, and automated starter template for Python projects at 42 School. It is designed to provide a clean architecture, strict dependency management, and built-in quality assurance tools right out of the box. 

Whether you are building a small script, a local server, or a complex machine learning project, this template guarantees that your development environment remains reproducible and your code meets the highest standards.

## 🏗️ Design Philosophy
This template was built with the following core principles:
- **Separation of Concerns (SRC Layout):** The application code is strictly separated from configuration files and tests. This prevents accidental imports, avoids packaging messy code, and enforces clean architecture.
- **Reproducibility:** By using the modern package manager **uv** alongside a strict **uv.lock** file, the environment is 100% identical on every machine. No more "it works on my machine" issues during evaluations.
- **Developer Experience (DX):** Everything is automated via a robust **Makefile**. You do not need to memorize complex commands to lint, test, or run your code.
- **Continuous Integration Ready:** The project includes a GitHub Actions workflow to automatically test and lint your code on every pull request.

## 🚀 Getting Started (New Project Checklist)
When you clone this template to start a brand new 42 project, make sure to complete this checklist before coding:

1. **Update pyproject.toml:** Open the file and change the **name** and **description** fields to match your new project.
2. **Update Makefile:** Open the Makefile and update the **NAME** variable at the top.
3. **Clean Demo Code:** if you don't need it : 
	- Delete **src/app/utils/math_helpers.py**
	- Delete **tests/test_math_helpers.py**
4. **Initialize Environment:** Run the setup command to build your virtual environment:
	```make install```
5. **Update README at the root of the repo:** Change the name of the project.

## 📦 Package Management with uv
This project uses **uv**, a blazing-fast Python package manager written in Rust. It replaces pip, virtualenv, and pip-tools all at once.

**Key Commands to remember:**

To add a standard dependency (e.g., numpy) that your program needs to run:
	
	uv add numpy

To add a development tool (e.g., a new linter) that is only needed for coding, not for production:

	uv add --dev ruff

To remove a package:

	uv remove numpy

To manually synchronize your environment with the uv.lock file (though the Makefile does this for you):

	uv sync

## 🧪 Testing with pytest
Testing is a mandatory skill. This template is pre-configured with **pytest**. 
To write tests, simply create files starting with **test_** inside the **tests/** directory. Inside these files, your testing functions must also start with **test_**.

**Key concepts and keywords:**

- ```assert```: The core of any test. It checks if a condition is true. If false, the test fails.

```python
assert add(2, 3) == 5
```

- ```pytest.raises```: Used to verify that your code correctly crashes and throws an error when given bad input (very important for 42 evaluations!).
```python
with pytest.raises(ValueError):
	divide(10, 0)
```

- ```pytest.fixture```: A way to provide a fixed baseline or setup data for your tests (like a dummy database connection or a pre-filled dictionary) that runs before your test functions.

## 🛠️ Makefile Reference
You do not need to call **uv** or **pytest** manually most of the time. Use the Makefile!

**Installation & Setup**
- ```make install``` (or ```make all```): Initializes the virtual environment and installs all dependencies.
- ```make setup```: Check the python version used on the current computer and check if `uv` is installed.

**Execution & Debugging**
- ```make run```: Executes the main entry point (src/app/main.py).
- ```make debug```: Launches the project using the Python Debugger (pdb).

**Quality & Testing**
- ```make lint```: Runs flake8 and mypy to ensure code style and type safety.
- ```make lint-strict```: Runs the linters with the strictest type-checking rules.
- ```make test```: Runs the entire test suite.
- ```make test ARGS="path/to/file.py"```: Runs a specific test file or uses specific pytest flags.

**Cleaning**
- ```make clean```: Removes temporary files and caches.
- ```make fclean```: Performs a deep clean, removing the virtual environment and build folders.
- ```make re```: Rebuilds the project from scratch.
